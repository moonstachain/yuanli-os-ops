# Self-hosted Dify Runbook

## 目标

把这套短视频内容系统运行在 `自托管 Dify` 上，供内部使用。

## 第一版环境建议

- 部署方式：`Docker Compose`
- 使用方式：内部 Web App
- 角色：单操盘者或少量内部成员
- 不开放外部访问

## 最小上线顺序

1. 按 Dify 官方 Docker Compose 文档部署基础环境
2. 配置工作空间、模型供应商、必要插件
3. 先创建 `选题雷达`
4. 再创建 `创作引擎`
5. 用表单或结构化输入建 `闭环进化`
6. 最后接 `主编排`

## 需要的外部能力

- `热点聚合接口`
  - 为 X 和海外 AI 媒体提供统一 JSON 输出
- `可选抓取服务`
  - 如果 Dify 内部直接抓不稳，单独做轻量服务供 HTTP Request 调用

## 第一版不要阻塞上线的事项

- 小红书自动发帖
- 视频号自动发帖
- 平台后台自动抓数
- 多账号矩阵

## 上线检查

- Dify 工作空间可用
- 模型配置完成
- 四个内部应用能分别独立运行
- `选题雷达` 能稳定输出 3 个候选
- `创作引擎` 能稳定输出脚本包
- `闭环进化` 能接受手动反馈并产出下一轮更新
- `主编排` 能维护状态流转

## 风险提示

- 热点抓取是第一版最不稳定的外部依赖
- 自动发平台不纳入 v1
- 若未来做自动抓数，平台授权和反爬策略会成为新复杂度来源
