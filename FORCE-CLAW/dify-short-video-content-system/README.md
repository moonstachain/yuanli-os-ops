# Dify Short Video Content System

这是一套面向 `原力创业` 新媒体短视频业务的 Dify 实现包。

目标不是一次性全自动，而是先把内容闭环稳定跑通：

1. `选题雷达`
2. `创作引擎`
3. `闭环进化`
4. `主编排`

## 第一版边界

- 部署目标：`自托管 Dify`
- 自动化边界：`前自动，后半自动`
- 热点来源：`X + 海外 AI 媒体`
- 发布平台：`小红书 + 视频号`
- 发布动作：人工完成
- 数据回收：人工录入或粘贴

## 目录

- `apps/`
  - 四个内部 Dify 应用的实现规格
- `contracts/`
  - 四类核心对象的 JSON Schema
- `dsl-drafts/`
  - 四个应用的 DSL-ready 蓝图
  - 这些文件用于导入前建模，不宣称可直接导入 Dify
- `templates/`
  - 选题选择、反馈录入示例
- `deployment/`
  - 自托管 Dify 运行说明与上线检查

## 建议落地顺序

1. 先在 Dify 中实现 `选题雷达`
2. 再实现 `创作引擎`
3. 用人工发布 + 手动录入方式跑通 `闭环进化`
4. 最后接上 `主编排`

## 成功标准

- 每轮稳定产出 `3 个候选选题`
- 选中 1 个后能生成 `可拍脚本 + 平台文案`
- 发布后能录入反馈并形成下一轮偏好修正
- 不依赖自动发帖或自动抓平台数据
