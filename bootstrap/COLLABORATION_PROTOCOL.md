# Collaboration Protocol

## What You Give

Only give these inputs by default:

- current battlefield
- current goal
- hard constraints
- human-only preferences or irreversible approval points

## What OS-原力 Owns

- `六判断`
- `主题层` entry decision
- `策略层` win path
- `执行层` routing
- verification
- evolution

## When OS-原力 Interrupts You

Only for:

- login or authorization
- irreversible external publish
- payment or budget approval
- subjective taste selection
- contradictory goals

## Default Calling Pattern

- normal complex work:
  - `AI大管家` first
  - then `AI大管家 -> OS-原力`
- explicit `3 × 3` method work:
  - call `OS-原力` directly
- system audit or maturity review:
  - `OS-原力` leads
  - add `AI大管家` only if closure sync or outer governance is needed

## Visible Output Standard

- `战略层`
  - whether the task is worth entering
  - how the task should win
  - what the current max distortion is
- `执行层`
  - what route is selected
  - what verification target applies
  - what reusable evolution was captured

## Dual-Machine Cutover Model

- old machine
  - rollback point
  - training ground
  - emergency fallback for missing auth or unexpected gaps
- new M5 machine
  - first-wave production machine
  - primary battlefield for `内容与增长`
  - target canonical environment after the `1-2` week overlap window
